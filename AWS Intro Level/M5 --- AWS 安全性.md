AWS 责任共担模式

AWS  Shared Responsibility Model

```
开发者和AWS双方需要共同合作以保护整个应用程序。

责任共担模式： AWS将开发者的应用程序堆栈作为一个整体来看，并将其分成多个不同的部分。其中一些部分由AWS 100% 负责，而其他部分由客户100% 负责。

堆栈：
	User Data （客户）
	Application （客户）
	Guest OS （客户）
	*** EC2 ***
	Hypervisor管理程序 （AWS）
	Network （AWS）
	Physical （AWS）
```

Identity and Access Management (IAM)

```
User: 在AWS IAM中，用户指的时永久命名操作员，可以是人，也可以是机器。 无论凭证是什么形式、无论它是名称和密码还是访问密钥或私密密钥组合，它都是永久的，并且始终与命名用户相关联。直到强制更换为止

Group： 组是用户的集合。组可以有很多用户，用户可以属于很多组。

Role: 在IAM中，角色不是权限。角色是一种身份验证方法。

用户是操作员，是一组永久凭证；角色也是操作员，但角色具有的凭证是临时的。

Policy Docs：在所有情况中，权限都发生在称为策略文档的单独对象中。策略文档是一个JSON文档，可以直接附加刀永久命名用户，或一组用户，也可以直接附加到角色。策略文档列出了列入白名单或允许针对某些资源使用的特定API或一组API通配符。
```

AWS Shield

```
是一种托管式分布式拒绝服务（DDoS）防护服务，可以保护在AWS上运行的应用程序。

DoS - 拒绝服务
让用户无法使用您的网站或应用程序
	- 占用带宽/系统资源从而使用户无法正常访问

DDoS - 分布式拒绝服务
使用多种来源来功击目标； 基础设施和应用程序层可能受到影响。

缓解DDoS功击面临的挑战：
	- 设置和实施复杂
	- 带宽限制
	- 手动干预
	- 耗时
	- 性能下降
	- 成本高
	
AWS Shield版本
	- Standatd 提供给所有AWS客户的自动保护服务，无需额外付费
	- Advanced 付费服务， 可以实现更高级别的保护、功能和优势
	
AWS Shield Standard：
	- 自动保护
		· 任何AWS资源
		· 任何AWS区域
	- 快速检测 --- 始终可用
	- 内联功击缓解
		· 内置自动化缓解技术
		· 避免延迟影响
	- 自动服务
		· 无需联系AWS Support

AWS Shield Advanced
	- 专业支持
	- 高级攻击缓解
	- 可见性攻击通知
	- 持续监控
		· Amazon Route 53， CloudFront， ELB， ELastic IP
	- 增强型检测
	- DDoS费用保护

优势：
	- 经济高效
	- 无缝集成和部署
	- 自定义防护

保护您的DNS：
	- 使用Route 53
		· Standard - 托管区域
		· Advanced - 攻击可见性，DRT支持
	
保护Web应用程序和API：
	- 使用CloudFront或Application Load Balancer
		· standard - 始终开启，清除不良流量
		· advancer - DRT支持、流量设计、应用程序层保护

保护其他应用程序:
	- 使用弹性IP地址
		· standard - 内置技术
		· advancer - 自定义缓解配置文件，额外的带宽
		
总结：
	- 提供的功能：
		· 内置DDoS攻击防护功能
		· 各种工具、服务和专业知识，帮助保护AWS应用程序
```

安全性与合规性

```
合规性方法：
AWS责任：
	· 提供高度安全并且受控的平台
	· 提供大量安全功能

客户责任：
	· 配置IT

AWS安全信息：
AWS通过以下方式共享安全信息：
	· 获得行业认证
	· 发部安全和控制实践
	· 合规性报告
	
保障计划：
	- AWS提供合规性信息和资源：
		· 认证/鉴证
		· 法律/法规支持
		· 协定/框架
	
风险与合规性计划
	- 帮助客户：
		· 记录完整的控制和监管框架
		· 部署复合多种行业特定标准的解决方案

合规性组成部分：
	· 风险管理
	· 控制环境
	· 信息安全
	
风险管理框架
	- AWS合规性与安全团队
		· 制定框架和政策
		· 维护安全策略
		· 提供安全培训
		· 执行应用程序安全审查
			- 数据的机密性、完整性和可用性
			- 与IS策略的一致性
			
风险管理过程：
	- AWS安全工具扫描漏洞，并通知相关方消除发现的漏洞
	- 客户请求权限以便对其云基础设施进行扫描
	- 独立安全公司定期执行漏洞威胁评估
	-  将调查结果/建议分类并提交给AWS领导层
	
控制环境
	- 包含策略、流程和控制活动，确保交付AWS服务产品
	- 支持AWS控制框架的运行有效性
	- 集成特定于云的控制措施
	- 应用领先的行业实践
	
信息安全：
	- 旨在保护
		· 机密性
		· 完整性
		· 可用性
	- 发部安全白皮书

客户合规性：
	- 查看受信任的信息， 并记录合规性要求
	- 设计并实施控制目标，满足合规性要求
	- 识别并记录外部各方拥有的控制体系
	- 验证所有控制目标是否均已达到，以及全部关键控制体系是否已设计并行之有效
	

```
